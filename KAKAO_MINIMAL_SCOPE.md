# 카카오 로그인 설정: 이메일만 사용

## 목표
- ✅ 이메일 (account_email) - 사용
- ❌ 프로필 사진 (profile_image) - 제외 (코드에서 사용 안 함)
- ❌ 닉네임 (profile_nickname) - 제외 (코드에서 사용 안 함)

## 주의사항
⚠️ **카카오는 프로필 정보(닉네임/프로필 사진)를 하나로 묶어서 제공합니다.**
- 프로필 정보는 카카오 로그인의 기본 제공 항목이지만, 코드에서 사용하지 않도록 처리했습니다
- 카카오 개발자 콘솔에서는 프로필 정보를 "선택 동의"로 설정하는 것을 권장합니다
- 이메일은 "필수 동의"로 설정해야 합니다

## 해결 방법

### 1. 카카오 개발자 콘솔 설정

1. [카카오 개발자 콘솔](https://developers.kakao.com/) 접속
2. 내 애플리케이션 → 앱 선택
3. 제품 설정 → 카카오 로그인 → 동의항목

4. **이메일 (account_email) 설정**
   - "설정" 버튼 클릭
   - **필수 동의**: 체크
   - **동의 화면 노출**: 체크
   - 저장

5. **프로필 정보(닉네임/프로필 사진) 설정**
   - "설정" 버튼 클릭
   - **필수 동의**: 체크 해제 (선택 동의로 변경)
   - **동의 화면 노출**: 체크 (사용자가 선택할 수 있도록)
   - 저장
   - ⚠️ 참고: 프로필 정보는 카카오에서 제공하지만, 코드에서 사용하지 않으므로 문제없습니다

### 2. 코드 설정 (이미 완료됨)

코드에서 이미 다음이 설정되어 있습니다:
- ✅ OAuth 콜백에서 프로필 생성 시 이메일만 사용
- ✅ 카카오에서 받은 닉네임/프로필 사진은 사용하지 않음
- ✅ 프로필의 `full_name`은 null로 설정 (나중에 사용자가 직접 입력)

### 방법 2: Supabase에서 이메일 없이 사용자 생성

Supabase는 기본적으로 이메일이나 전화번호가 필요하지만, 카카오 로그인 사용자의 경우:
- 이메일이 없어도 사용자 계정을 생성할 수 있습니다
- 프로필 생성 시 이메일을 null로 처리

### 주의사항

⚠️ **중요**: 
- Supabase는 이메일이나 전화번호가 없으면 일부 기능이 제한될 수 있습니다
- 비밀번호 재설정 등 이메일 기반 기능을 사용할 수 없습니다
- 프로필에서 이메일이 표시되지 않을 수 있습니다

### 현재 설정

현재 코드는 카카오 개발자 콘솔의 동의 항목 설정에 따라 자동으로 요청합니다.
- 이메일 동의 항목을 비활성화하면 이메일을 요청하지 않습니다
- 프로필 정보는 필수이므로 유지됩니다

## 테스트

1. 카카오 개발자 콘솔에서 이메일 동의 항목 비활성화
2. 5-10분 대기 (설정 반영 시간)
3. 카카오 로그인 버튼 클릭
4. 정상적으로 로그인되는지 확인
