# 카카오 로그인 KOE205 에러 해결 방법

## 에러 내용
```
잘못된 요청 (KOE205)
설정하지 않은 카카오 로그인 동의 항목을 포함해 인가 코드를 요청했습니다.
설정하지 않은 동의 항목: account_email, profile_image, profile_nickname
```

## 해결 방법

### 1. 카카오 개발자 콘솔 접속
1. [카카오 개발자 콘솔](https://developers.kakao.com/) 접속
2. 로그인
3. 내 애플리케이션 → "다애교회" 앱 선택

### 2. 동의 항목 설정
1. 왼쪽 메뉴에서 **제품 설정** 클릭
2. **카카오 로그인** 클릭
3. **동의항목** 탭 클릭

### 3. 필수 동의 항목 활성화
다음 항목들을 **반드시 활성화**해야 합니다:

#### ✅ 닉네임 (profile_nickname)
- **필수 동의**: 체크
- **동의 화면 노출**: 체크
- **이용 목적**: "서비스 이용을 위한 닉네임 사용" 등 입력

#### ✅ 이메일 (account_email)
- **필수 동의** 또는 **선택 동의**: 체크 (선택)
- **동의 화면 노출**: 체크
- **이용 목적**: "서비스 이용을 위한 이메일 사용" 등 입력

#### ✅ 프로필 사진 (profile_image)
- **필수 동의** 또는 **선택 동의**: 체크 (선택)
- **동의 화면 노출**: 체크
- **이용 목적**: "프로필 사진 표시" 등 입력

### 4. 저장 및 확인
1. 각 항목 설정 후 **저장** 버튼 클릭
2. 변경사항이 반영되는지 확인 (몇 분 소요될 수 있음)

### 5. 다시 테스트
1. 앱에서 카카오 로그인 버튼 클릭
2. 정상적으로 카카오 로그인 화면이 표시되는지 확인

## 주의사항

- 동의 항목을 활성화한 후 **즉시 반영되지 않을 수 있습니다** (최대 5-10분 소요)
- 카카오 앱이 **심사 중**이거나 **비활성화** 상태면 동의 항목 설정이 제한될 수 있습니다
- **필수 동의**로 설정하면 사용자가 반드시 동의해야 하며, **선택 동의**로 설정하면 사용자가 선택할 수 있습니다

## 참고

- 동의 항목 설정은 카카오 정책에 따라 변경될 수 있습니다
- 더 자세한 내용은 [카카오 개발자 문서](https://developers.kakao.com/docs/latest/ko/kakaologin/rest-api)를 참고하세요
